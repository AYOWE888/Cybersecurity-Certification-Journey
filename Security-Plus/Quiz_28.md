# CompTIA Security+ (SY0-701) Quiz Report: Domain 4.2 & 4.3

---

### Key Points Tested

* **Asset Management (4.2):** Tracking physical property (**CapEx** with depreciation) vs. licenses/subscriptions (**OpEx**). Data sanitization techniques including overwriting for drive reuse, **degaussing** to destroy magnetic media properties, and physical shredding verified via a **Certificate of Destruction**. Archiving timelines are defined by a **Data Retention Policy**.
* **Vulnerability Management (4.3):** Comparing **active scans** (sending intrusive probes) to **passive scans** (silently monitoring traffic). **Credentialed scans** log in with admin privileges to audit registries and patches, lowering false positives, while **non-credentialed scans** check external perimeters.
* **Results & Prioritization:** Distinguishing **false positives** (safe systems flagged as vulnerable) from **false negatives** (missed real threats). Flaws are prioritized using **CVSS** severity tiers and treated via permanent **remediation** (patching) or compensating **mitigation** (firewall/IPS blocks).
* **Penetration Testing:** Operations governed by **Rules of Engagement (RoE)** using **Black-box** (zero-knowledge), **Gray-box** (partial user access), or **White-box** (full source/diagram access) approaches. Testers execute **pivoting** to move laterally from a low-security foothold into isolated subnets, and establish **persistence** across reboots.
* **Simulation Teams:** **Red Team** acts as the offensive adversary, **Blue Team** defends, **White Team** acts as referee, and **Purple Team** bridges them through real-time collaboration.

| # | Core Concept | Scenario Breakdown | Key Technical Takeaway |
| --- | --- | --- | --- |
| **1** | **Degaussing** | Destroying regulatory data while rendering physical platter drives completely unusable. | A strong electromagnetic field permanently disrupts magnetic domains, breaking both data and drive servo tracks. |
| **2** | **CapEx vs. OpEx** | Financial tracking differences between server hardware and software licensing. | Hardware is a capital expenditure (**CapEx**) written off over time; software licenses are running operational expenses (**OpEx**). |
| **3** | **Credentialed Scan** | Evaluating internal registry keys, patch verification lists, and local folder permissions. | Requires local administrative login rights to inspect internal host file systems and deep configurations. |
| **4** | **Black-Box Testing** | Simulating an engagement with zero architectural maps, IP ranges, or credential access. | Replicates a real-world outside adversary entering the target environment with zero foundational knowledge. |
| **5** | **Mitigation** | Deploying an IPS in front of an unpatchable legacy system to block exploit strings. | A compensating security block that reduces exploit risk without fixing the underlying unpatched code error. |
| **6** | **Pivoting** | Using a compromised public-facing server to run scans against an isolated internal database. | The tactical process of leveraging a breached node as an inside proxy to hop deeper into segmented zones. |
| **7** | **False Positive** | A scanner alerts on a critical application vulnerability that was verified as patched. | An incorrect flag generated by an evaluation engine on a baseline that is functionally secure. |
| **8** | **Purple Team** | Fusing offensive attacker models with defensive monitor teams to enhance infrastructure. | Promotes open, collaborative loop feedback and data sharing between the **Red** and **Blue** teams. |
